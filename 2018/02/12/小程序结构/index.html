小程序结构

发表于 2018-02-12 | 分类于 2018-02

初稿 2018.02.12
修改 2018.04.03

在创建了一个带模板的小程序项目之后，在项目里会看见不同类型的文件。这些文件分别是

  1. .json 后缀的 JSON 配置文件

  2. .wxml 后缀的 WXML 模板文件

  3. .wxss 后缀的 WXSS 样式文件

  4. .js 后缀的 JS 脚本逻辑文件

初看结构十分类似 HTML + CSS + JS 三剑客，进一步了解后他们的作用也大抵相同。在小程序总体结构上，分成两部分。一部分是全局文件，都是以app开头，承载整个小程序的全局信息,例如 app.json, app.wxss, app.js。另一部分是局部文件，具体到每个一个页面都拥有自己的一套配置信息，局部文件的优先级会大于全局文件。例如 index.wxml, index.wxss, index.js, index.json

下面我们以这样一个目录结构来依次介绍

先忽略掉 typinggs 目录。

JSON配置文件

从项目根目录，我们依次看到 app.json, project.config.json 和 index.json。

project.config.json

project.config.json是对于微信开发者工具一些个性化配置。当你换了一台电脑或者重新安装工具时，你只要载入同一个项目的代码。开发者工具就会根据这些配置进行恢复。

app.json

app.json 是对当前小程序的全局配置，必须要传入一个对象参数，空对象也可以。里面有几个配置项，分管不同的功能。

配置项有：

  • pages 注册页面的路径
  • window 管理默认页面的窗口表现
  • tabBar 设置底部tab的表现
  • networkTimeout 设置网络超时时间
  • debug 设置是否开启debug模式

WXML WXSS JS

它其实类似 HTML + CSS + JS 这个，不过他们之间都有一些细微的差距。

其实只需要知道，WXML是用来组织页面的结构的，WXSS是用来添加样式的，JS是对应页面的逻辑。

需要注意的是，对于一套页面来说，它必须有相同名称。举一个理解，要创建一个index页面，那么index.wxml 代表它的组织结构，index.wxss代表它的样式信息，index.js 代表它的逻辑信息。

微信小程序会自动通过命名来找到页面的相关配置文件。

对于 page.js 和 app.js 里面必须对应调用 page() 和 app() 来注册一个页面或者app。 通过传入的对象参数来对其进行控制。
